# 美股投研 Skill 中文说明

这份文件是 `SKILL.md` 的中文维护版，主要给你自己检查、修改、升级这个 Skill 时使用。正式触发给 Codex / Agent 的文件是英文版 `SKILL.md`。

## 目标

这个 Skill 用于分析你已经选定的美股上市公司，服务于中期和长期投资研究。默认所有股票都进入深度个股研究；如果问题和财报、季报、指引、10-Q、股东信、业绩电话会有关，则进入财报后追踪模式。

默认支持：

- 对选定股票输出深度个股研究。
- 财报后追踪公司变化。
- 重新评估 thesis、估值、风险和后续跟踪点。
- 只有在假设和来源清楚时，才估算估值区间。

默认周期：

- 中期：几周到几个月。
- 长期：一年以上。

## 禁止事项

必须遵守：

- 不编造财务数据、估值倍数、管理层表述、业绩指引、分析师评级或新闻。
- 涉及当前价格、最新财报、估值时，必须核实最新信息。
- 需要引用事实来源，优先使用公司 IR、SEC 文件、财报电话会、正式公告。
- 区分事实、估算、假设和观点。
- 不给没有假设依据的目标价、公允价值或估值区间。
- 不能对所有公司使用同一种估值方法。估值方法必须匹配公司阶段、盈利能力、周期性和行业属性。
- 不能直接相信 adjusted EBITDA、non-GAAP EPS 或自由现金流，必须检查 SBC、营运资本、资本化成本和一次性项目。
- 不能忽略下行情景、稀释、债务、竞争、利润率、监管、客户集中度、管理层可信度等风险。
- 输出应是投研辅助和决策框架，不应包装成个性化投资建议。

## 资料优先级

优先使用：

1. 公司 IR：财报、股东信、演示文稿、10-K、10-Q、8-K、业绩指引。
2. SEC 文件和正式电话会纪要。
3. 市场数据：股价、市值、估值倍数、历史表现。
4. 可靠财经新闻，用于解释近期催化。
5. 分析师评级变化只能作为辅助，不作为核心依据。
6. 行业数据和同行财报，用于比较增长、利润率和估值。

如果关键资料缺失，需要说明缺什么，并降低结论置信度。

## 数据核验清单

在给 thesis、估值区间或财报后观点前，必须核实并说明日期或期间：

- 当前股价、市值、企业价值和稀释后股本。
- 最新财季和 TTM 财务数据。
- 收入增长、毛利率、经营利润率、净利润或亏损、自由现金流、现金余额。
- 债务、租赁负债、到期结构、净现金或净债务。
- SBC、股本稀释、回购和重大股权融资。
- 最新指引、可获得时的市场预期背景、管理层表述。
- 相关时，下一次财报日期或重要催化。

如果数据不可得或来源不一致，要标明不确定性，不要强行精确。

## 模式选择

只支持两种模式：

- **深度个股研究**：所有股票代码或公司名分析请求默认使用，即使用户只给了一个 ticker。
- **财报后追踪**：用户提到 earnings、财报、最新季报、guidance、10-Q、股东信、投资者演示文稿、业绩电话会时使用。

不要再设计其他独立模式。若用户提出其他形式的问题，应归入最接近的两种模式之一。

## 保存研究结果

除非用户明确要求不要保存，否则研究结果要保存为 Markdown 文件。每家公司单独一个目录：

```text
research/
└── COMPANIES/
    └── TICKER/
        ├── deep-research/
        │   ├── YYYY-MM-DD-deep-research.md
        │   └── YYYY-MM-DD-deep-research-v2.md
        ├── earnings-tracking/
        │   ├── YYYY-QN-earnings-tracking.md
        │   └── YYYY-MM-DD-earnings-tracking.md
        └── tracker.md
```

公司目录使用大写 ticker，例如 `LMND`、`NVDA`、`PLTR`。

### deep-research/

完整深度个股研究保存到 `research/COMPANIES/TICKER/deep-research/`。

不要覆盖旧的深度研究文件。每篇深度研究都代表某个时间点的完整 thesis 快照。只有当公司逻辑发生明显变化、需要重做完整研究时，才新增一版。

### earnings-tracking/

每次财报后追踪保存到 `research/COMPANIES/TICKER/earnings-tracking/`。

如果财季明确，优先使用 `YYYY-QN-earnings-tracking.md`。如果财季不明确，或不是常规财报更新，则使用 `YYYY-MM-DD-earnings-tracking.md`。

### tracker.md

`research/COMPANIES/TICKER/tracker.md` 是当前状态文件。每次完成深度研究或财报后追踪后，都要更新它。

`tracker.md` 应总结：

- 当前 research view。
- 当前 main thesis。
- 关键风险。
- 最新财报或公司更新。
- 相比上次观点发生了什么变化。
- 下一步要跟踪的指标或事件。
- 什么情况会改变研究观点。

### 更新规则

- 小变化：只更新 `tracker.md`。
- 财报变化：新增 `earnings-tracking/...md`，再更新 `tracker.md`。
- 公司逻辑大变：新增一版 `deep-research/...-v2.md`，再更新 `tracker.md`。

## 核心研究流程

每次非简单分析都要：

1. 确认公司、股票代码、交易所、行业和商业模式。
2. 核实当前股价、市值、近期表现和最新财报期。
3. 如果分析依赖当前事实，阅读最新公司材料和相关新闻。
4. 先判断公司类型，再选择估值方法。
5. 基于商业质量、增长驱动、财务趋势、估值和催化形成初始 thesis。
6. 列出支持证据、反证证据和最大不确定性。
7. 压测下行风险、稀释、资产负债表风险和估值倍数压缩。
8. 给出研究观点、置信度、时间周期、风险收益和改变 thesis 的条件。
9. 按“保存研究结果”规则保存输出，并更新 `tracker.md`。

## 公司类型适配

根据公司类型调整研究和估值方法：

- **高成长但未盈利的软件 / 互联网公司**：关注收入耐久性、毛利率、NRR、获客效率、经营杠杆、现金消耗、SBC、稀释、EV/Sales 或通往 FCF 的路径。
- **盈利型复利公司**：关注收入增长、利润率耐久性、ROIC、自由现金流转化、再投资空间、竞争优势、P/E、EV/EBIT、EV/EBITDA 或 FCF yield。
- **周期股**：用跨周期收入、利润率和估值倍数进行归一化，不能把景气顶点利润当成常态利润。
- **金融 / 保险 / 金融科技信贷公司**：关注信用质量、赔付率、融资成本、资本充足率、监管风险、账面价值、ROE 和准备金充足性。
- **生物科技 / 医疗创新公司**：关注管线阶段、临床里程碑、成功概率、现金跑道、稀释、监管路径和潜在市场。
- **困境反转公司**：关注流动性、债务到期、契约风险、现金消耗、资产出售和反转是否可量化。

## 研究观点框架

研究观点必须一致使用：

- **Positive**：thesis 有基本面支撑，估值在合理假设下说得通，指定周期内风险收益有吸引力。
- **Constructive but watchful**：公司有吸引力，但估值、时点、波动或缺少确认信号，需要继续验证。
- **Neutral / wait for confirmation**：公司值得跟踪，但当前证据不足以形成高置信度 thesis。
- **Negative / thesis weakening**：基本面、估值、稀释、资产负债表风险或竞争地位正在削弱 thesis。
- **Insufficient evidence**：关键数据缺失、来源冲突，或 thesis 需要过多猜测时使用。必须说明还需要什么证据，才能升级成方向性观点。

给任何研究观点时，都必须同时写出反对这个观点的最强理由。

## 估值纪律

必须说明为什么所选估值方法适合这家公司：

- EV/Sales 或 EV/Gross Profit 主要用于高成长或未盈利公司，且必须连接收入增长、长期利润率和现金消耗。
- P/E、PEG、EV/EBITDA、EV/EBIT 或 FCF yield 用于盈利公司，并调整周期性、一次性项目和盈利质量。
- 只有在能清楚说明假设时才使用 DCF；要写收入增长、利润率、FCF 转化、折现率、永续增长率或终值倍数、敏感性。
- 对金融和保险公司，相关时使用账面价值、有形账面价值、ROE、综合成本率、赔付率或信用指标。
- 重要结论必须做熊 / 基准 / 牛三种情景估值。每个情景要有假设、隐含价值和触发条件。
- 重大时必须纳入稀释和 SBC。优先使用稀释股本和企业价值，不要只做简单市值估算。

## 风险收益框架

深度研究和财报后追踪都要包括：

- 数据允许时，给出基准上行空间和下行情景。
- 牛市情景需要哪些条件成立。
- 什么会打破投资逻辑。
- 风险主要来自估值、执行、资产负债表、竞争、监管、宏观还是时点。
- 下一季度或下一次重大更新要重点跟踪什么。

## 深度个股研究模板

所有选定股票默认使用：

开头固定使用这个摘要块：

- **Research view**：
- **Confidence**：
- **Time horizon**：
- **Main thesis**：
- **Key risk**：

1. **执行摘要**
   - 研究观点：Positive / Constructive but watchful / Neutral / Negative / Insufficient evidence。
   - 置信度和适合周期。
   - 三条核心投资逻辑。
   - 反对该观点的最强理由。

2. **商业模式与行业结构**
   - 公司如何赚钱。
   - 可获得时分析单位经济模型。
   - 市场规模、渗透率、增长驱动因素。
   - 竞争格局和护城河。
   - 客户集中度、定价权、切换成本。

3. **分部和产品分析**
   - 可获得时拆分收入分部或产品线。
   - 各分部增长、利润率和战略重要性。
   - 产品路线图或创新管线。

4. **财务深挖**
   - 多年收入增长和增长质量。
   - 毛利率、经营利润率、经营杠杆。
   - 自由现金流和资本开支强度。
   - 资产负债表、债务期限、现金跑道、稀释风险。
   - 关键会计问题或 non-GAAP 调整。
   - SBC、股本变化、营运资本和一次性项目。

5. **管理层和资本配置**
   - 管理层可信度和执行记录。
   - 内部人持股或激励。
   - 回购、稀释、并购、资本开支、研发投入。

6. **估值分析**
   - 当前估值倍数和历史位置。
   - 有意义时做同行比较。
   - 熊 / 基准 / 牛三种情景估值。
   - 明确收入增长、利润率、估值倍数、折现率或终值等假设。
   - 按公司类型选择估值方法。

7. **催化剂和跟踪计划**
   - 短期催化。
   - 中期投资逻辑里程碑。
   - 未来几个季度要跟踪的指标。

8. **风险登记表**
   - 关键风险的概率和严重性。
   - 下行情景。
   - 投资逻辑失效信号。
   - 最强反证证据。

9. **最终研究框架**
   - thesis 依赖什么。
   - 什么情况会让 thesis 更强。
   - 什么情况会改变研究观点。

## 财报后追踪模板

适用于最新财报、earnings、业绩指引、10-Q、股东信、投资者演示文稿、业绩电话会：

开头固定使用这个摘要块：

- **Research view**：
- **Confidence**：
- **Time horizon**：
- **Main thesis**：
- **Key risk**：

1. **财报后结论**
   - 好于预期 / 混合 / 差于预期。
   - 更新后的研究观点：Positive / Constructive but watchful / Neutral / Negative / Insufficient evidence。
   - 核心原因。
   - 本季度是否改变长期投资逻辑。

2. **关键数据**
   - 收入、增长、EPS 或净利润、利润率、现金流。
   - 和去年同期、上季度、指引、市场预期比较。
   - 区分 GAAP / 报告结果和 non-GAAP / 调整后结果。

3. **指引和管理层表述**
   - 新指引。
   - 管理层重点表述。
   - 和之前叙事相比有什么变化。
   - 管理层语气或战略优先级是否变化。

4. **财报质量**
   - 增长是否健康，还是一次性因素。
   - 利润率趋势。
   - 现金流和资产负债表。
   - 客户、产品、分部信号。
   - 相关时检查行业 KPI，例如 ARR、RPO、NRR、GMV、take rate、bookings、backlog、loss ratio、combined ratio、同店销售、临床里程碑。

5. **市场反应背景**
   - 判断财报后市场反应是否与基本面、指引和估值变化一致。
   - 只在服务财报结论时区分基本面、情绪、仓位和估值重定价。

6. **更新后的 thesis 和跟踪计划**
   - 什么变好了。
   - 什么变差了。
   - 哪些假设改变了。
   - 下次财报前要跟踪什么。
   - 什么情况会改变研究观点。

## 输出风格

- 先给研究观点，不要长篇铺垫。
- 匹配用户语言：用户用中文提问时用中文输出，用户用英文提问时用英文输出。正式章节标题要保持清晰一致。
- 表格只在比较和假设展示更清楚时使用。
- 深度研究要完整，但每一节都要围绕对研究结论有用的证据。
- 财报、文件、价格、新闻等涉及日期的信息要标明日期。
- 关键事实要给来源。
- 结尾必须有简洁的 “What would change my view / 什么会改变我的观点”。
