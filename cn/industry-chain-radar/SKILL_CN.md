# 产业链爆发雷达（中文版）

## Overview / 总览

这个 skill 用于做**美股机会发现**，但不是从普通股票筛选器开始，而是从已经被市场验证的行业或龙头股开始，沿着产业链寻找下一层瓶颈、第二波受益者、第三波隐藏供应商和下一批值得深度研究的公司。

核心模式是：

1. 某个龙头股或行业已经爆发，证明需求真实存在。
2. 市场开始追问：龙头增长会制造哪个瓶颈？
3. 第二层供应链公司开始重估。
4. 第三层供应商、基础设施、能源、服务或隐藏受益者可能接着被发现。
5. 最终输出：哪些环节已经爆发，哪些处于第二波确认，当前最值得关注的下一环是什么，哪些公司应该进入 `us-equity-research` 深研。

典型例子：

```text
AI训练/推理需求爆发
→ GPU/ASIC需求爆发，NVDA验证主趋势
→ HBM/DRAM/存储成为瓶颈，MU、WDC、STX等受益
→ 高速互联和光模块需求上升，ANET、MRVL、COHR、LITE、AAOI、CIEN等进入视野
→ 数据中心电力、散热、施工、电网升级成为下一瓶颈，VRT、GEV、PWR、ETN、EME、HUBB等受益
→ 发电资产、核电、天然气、电网资产再成为更后面的传导层
```

本 skill 的目标不是回答“今天哪只股票涨得多”，而是回答：

> 哪个行业已经被龙头验证？这个增长会压垮哪一环？谁卖这个瓶颈？哪些公司还没有完全被市场定价？

## When to Use / 什么时候使用

当用户提出以下需求时使用：

- “下一轮爆发行业是什么？”
- “某个龙头已经涨完了，后面产业链谁受益？”
- “AI / GLP-1 / 军工 / 核电 / 电力 / 数据中心后面还有什么链条？”
- “帮我找美股产业链二阶受益者 / 隐藏供应商 / 下一波机会。”
- “不要简单选股，要解释为什么上一环爆发会带动下一环。”
- 需要保存行业雷达报告到 `/home/ubuntu/stock-reports/cn/industry-chain-radar/`。

不要把这个 skill 用作单一公司的深度分析。用户只问某个具体 ticker 的深研时，应使用 `us-equity-research`。

## 用户偏好的输出形态

用户不想要：

- 一堆股票代码；
- 泛泛而谈的宏观叙事；
- 只有涨幅排名的动量筛选；
- “AI很火，所以买相关股”这种跳跃式结论。

用户想要的是：

> **第一环为什么爆了 → 为什么下一环可能爆 → 代表标的是谁、做什么 → 最终表格一眼看懂。**

因此，每次输出必须围绕**因果链条**，而不是围绕股票清单。

## Mandatory Output Template / 必须输出模板

每份正式回答或保存的报告必须按以下顺序组织：

### 1. 一句话结论

直接说明：当前最可能进入下一波的产业链环节是什么，为什么。

示例：

> 当前更值得关注的是“AI数据中心电力与散热”这一环，因为GPU需求已经验证，集群规模扩大后最先暴露的不是芯片需求，而是供电容量、机房热管理和电网接入瓶颈。

### 2. 爆发链条总览

用箭头画出完整链条：

```text
AI训练/推理需求爆发
→ GPU/ASIC
→ HBM/先进封装
→ 光模块/交换机
→ 电力/散热/数据中心施工
→ 电网/发电资产
```

完成标准：用户只看这条箭头链，也能理解“为什么从A传到B，再传到C”。

### 3. 第一环为什么已经爆发

必须解释已经爆发的一环为什么是真需求，而不是纯炒作。证据包括：

- 龙头收入加速；
- backlog / bookings / RPO 增长；
- 管理层上调指引；
- 客户 capex 扩张；
- 产能短缺或交期变长；
- 毛利率或价格提升；
- 股价和行业篮子相对 SPY/QQQ 明显跑赢。

完成标准：列出龙头或行业已经爆发的具体证据，并判断它是“信号源”还是仍有早期机会。

### 4. 第二环 / 下一环为什么可能爆发

必须写出机械传导，而不是只写“相关”。使用这个句式：

```text
谁的需求增加
→ 哪个瓶颈出现
→ 谁卖这个瓶颈
→ 订单、收入、价格或利润如何体现
```

示例：

```text
AI集群规模扩大
→ 单机柜功率密度上升、散热难度上升
→ 液冷、电源管理、UPS、数据中心施工和电网接入变成瓶颈
→ VRT、ETN、PWR、EME等公司可能通过订单、backlog和利润率体现受益
```

完成标准：每个下一环必须能回答“为什么是这个瓶颈，而不是随便找一个相关概念”。

### 5. 代表标的解释

代表标的只能在链条逻辑讲清楚之后出现。每个 ticker 至少说明：

- 公司是做什么的；
- 它位于产业链哪一环；
- 收入或订单与主题的关系；
- 为什么可能受益；
- 需要验证什么数据；
- 为什么可能失败。

最小格式：

```text
- VRT：做数据中心电源、热管理和基础设施设备，位于AI数据中心电力/散热环节；如果AI机柜功率密度继续上升，订单和利润率可能受益；需要验证backlog、订单增速、毛利率和估值是否已经透支。
```

### 6. 阶段判断

必须把每一环分到以下五类之一：

1. **已经爆发**：主龙头或主链条已经被大幅重估，只能作为信号源，不能假装早期发现。
2. **第二波确认**：相关公司已经动了，但如果财报继续确认，可能还有第二腿。
3. **当前下一环焦点**：最值得当前研究的瓶颈或供应链层。
4. **早期观察**：逻辑成立但数据、价格或订单尚未确认。
5. **概念风险 / 避免追高**：主要靠叙事上涨，缺少收入、订单或利润验证。

完成标准：不能把所有标的都说成“机会”。必须明确哪些已经太拥挤、哪些只是观察。

### 7. 最终直观总表（强制）

每份报告必须以一张表帮助用户比较：

| 产业链环节 | 爆发原因 / 传导逻辑 | 代表标的 | 公司是做什么的 | 当前阶段 | 需要验证的数据 | 主要风险 | 下一步动作 |
|---|---|---|---|---|---|---|---|
| GPU/ASIC | AI训练和推理需求直接拉动 | NVDA/AMD/AVGO | 加速芯片和定制芯片 | 已经爆发 | 数据中心收入、订单、毛利 | 估值高、客户capex放缓 | 作为信号源 |
| 电力/散热 | 集群规模扩大导致机柜功率和热管理瓶颈 | VRT/ETN/PWR | 电源、散热、电网工程 | 当前下一环焦点 | backlog、订单、margin | 估值透支、项目延迟 | 深研Top候选 |

完成标准：用户只看箭头链和最终表格，也能理解为什么上一环爆、下一环为什么可能爆、每家公司到底卖什么、下一步该查什么。

### 8. 下一步深研清单

列出 3–5 个最值得用 `us-equity-research` 深研的 ticker，并按优先级排序。

排序依据：

- 在产业链中是否是关键瓶颈；
- 是否已经有订单、收入、backlog或margin验证；
- 估值不确定性是否足够大，值得深研；
- 是否符合用户组合规则和风险承受能力。

不要把雷达候选直接说成可以买。雷达只负责发现方向；买入判断必须进入公司级深研。

## Workflow / 工作流

### 1. 识别已验证主趋势

从已经爆发的龙头或行业开始，查找：

- 收入加速；
- backlog / bookings / RPO；
- 指引上调；
- 大客户 capex 扩张；
- 产能短缺、交期变长；
- 毛利率扩张；
- 相对 SPY/QQQ 跑赢；
- 行业宽度是否扩大。

完成标准：说明谁已经爆发、为什么是真需求、是否已经拥挤。

### 2. 画出产业链地图

每个趋势都要拆成：

- 终端需求；
- 核心产品；
- 关键供应商；
- 瓶颈零部件；
- 制造设备；
- 基础设施；
- 能源、电网、物流、数据、安全、服务；
- 每一环代表公司。

完成标准：每家公司必须被绑定到一个具体功能，不允许只写“与AI相关”。

### 3. 问瓶颈问题

对每条链问：

- 如果龙头需求翻倍，最先卡在哪里？
- 哪个输入供给有限？
- 哪个环节认证周期长？
- 哪个供应商有定价权？
- 哪个服务必须先扩容，主趋势才能继续？
- 哪些公司的收入会滞后一到两个季度才体现？

完成标准：找出 1–3 个最可能成为下一波的瓶颈环节。

### 4. 检查市场传导

尽量使用当前数据检查：

- 1个月、3个月、6个月涨跌幅；
- 相对 SPY/QQQ 和行业ETF表现；
- 距离52周高点；
- 龙头和二线公司是否一起动；
- 小型或隐藏供应商是否开始补涨；
- 成交量和行业宽度是否改善。

完成标准：把每个环节归类为已爆发、第二波确认、下一环焦点、早期观察或概念风险。

### 5. 检查基本面确认

对高优先级候选，检查：

- 相关主题收入占比；
- backlog / bookings / RPO / book-to-bill；
- 管理层是否明确提到需求驱动；
- 指引是否上调；
- 毛利率和营业利润率；
- capex和产能扩张；
- 客户集中度和合同期限；
- 供给约束和价格能力。

完成标准：明确写出“已确认 / 部分确认 / 仍偏投机”。

### 6. 保存雷达报告

如果用户要求保存报告，使用路径：

```text
/home/ubuntu/stock-reports/cn/industry-chain-radar/YYYY-MM-DD-next-wave-industry-chain-radar.md
```

报告必须包括：

- 一句话结论；
- 爆发链条总览；
- 已爆发行业；
- 当前下一环焦点；
- 代表公司解释；
- 阶段分类；
- 触发条件；
- 失败条件；
- 最终总表；
- 下一步深研清单。

完成标准：报告必须清楚说明哪些行业已经不是早期、哪些行业当前最值得关注、下一步深研哪些公司。

## Sector Playbooks / 行业剧本

### AI基础设施

典型传导：

```text
GPU/加速器
→ 晶圆代工/半导体设备
→ HBM/DRAM/存储
→ 高速互联/交换机/光模块
→ 数据中心电力/散热/施工
→ 电网/发电资产/核电/天然气
→ 软件/安全/应用变现
```

代表层级：

- GPU/ASIC：NVDA、AMD、AVGO。
- 代工和设备：TSM、ASML、AMAT、LRCX、KLAC。
- HBM/DRAM/存储：MU、WDC、STX、SNDK及非美股同行。
- 网络/光模块：ANET、MRVL、AVGO、COHR、LITE、AAOI、CIEN。
- 电力/散热/数据中心基础设施：VRT、ETN、GEV、PWR、EME、HUBB、GRID。
- 发电/核电/电网：CEG、VST、NEE、CCJ、UEC、URA、BWXT。

关键问题：当前AI最强链条之后，哪个物理或经济瓶颈会限制下一轮扩张？

### GLP-1 / 减肥药

主龙头：LLY、NVO。

潜在传导：

- 注射笔和给药设备；
- CDMO和灌装产能；
- 包装和冷链物流；
- 药房和分销；
- 诊断、随访和慢病管理；
- 潜在输家：胰岛素、透析、肥胖相关手术/器械、部分食品链。

关键问题：哪些公司有真实GLP-1收入暴露，而不只是“减肥药概念”？

### 军工 / 航空航天 / 无人机

驱动因素：

- 国防预算；
- 导弹和拦截弹需求；
- 无人机和反无人机；
- 雷达、传感器、电子战；
- 航空发动机和零部件供应链。

候选层：

- 大型主承包商：LMT、RTX、NOC、GD。
- ETF/行业宽度：ITA、XAR、PPA。
- 零部件供应商：HWM等航空供应链公司。
- 高风险中小票：KTOS、AVAV及相关无人机/电子战公司。

关键问题：订单和backlog是否开始从主承包商传导到小型供应商？

### 核电 / 电力

潜在层级：

- 电力运营商和发电资产：CEG、VST、NEE。
- 铀：CCJ、UEC、URA。
- 核设备和服务：BWXT。
- SMR高风险概念：SMR及同类公司。
- 电网和电气设备：GRID、ETN、HUBB等。

关键问题：市场买的是直接发电现金流，还是已经轮动到核燃料、设备和SMR概念？

## Trigger Conditions / 升级条件

一个主题可升级时，通常出现多个信号：

- 主龙头继续上调指引；
- 大客户capex继续增加；
- 瓶颈供应商订单加速；
- 毛利率扩张而不是收缩；
- 主题篮子1–3个月跑赢SPY/QQQ；
- 二线公司开始跑赢主龙头；
- 以前不被关注的供应商开始在财报电话会提到该主题；
- 分析师预期在公司证据之后上调，而不是先靠故事上调。

## Failure Conditions / 失败条件

主题应降级时：

- 主龙头miss指引或客户capex放缓；
- 发现所谓瓶颈并不稀缺；
- 收入没有传导到二阶供应商；
- 竞争或过剩导致毛利率压缩；
- 涨幅只扩散到低质量概念票，而核心公司走弱；
- 估值要求永久高增长或永久峰值利润率；
- 稀释、债务或资产负债表风险压倒行业逻辑。

## Radar-to-Deep-Research Handoff / 从雷达到深研

雷达只负责发现方向，不负责直接下买入结论。

当某个候选进入Top 3–5后，下一步应使用 `us-equity-research` 对单家公司深研，并检查：

- 公司到底卖什么；
- 主题收入占比；
- backlog / RPO / book-to-bill；
- 毛利率和经营杠杆；
- capex和现金流；
- SBC、债务和稀释；
- 熊/基准/牛估值；
- 当前价格是否已经透支；
- 是否符合用户组合规则。

对工业基础设施公司，还要特别检查订单质量、项目执行风险、保修/质量准备金、营运资本、资本开支和一次性收益，避免把周期高点利润当成正常利润。

## Common Pitfalls / 常见错误

1. **从涨幅筛选开始。** 涨幅只能作为确认信号，不能作为 thesis 来源。
2. **把已经爆发的链条说成早期。** 已经翻倍或数倍的篮子只能归类为已爆发或第二波确认。
3. **把叙事暴露当收入暴露。** 公司必须卖出瓶颈所需的真实产品或服务。
4. **忽略时间滞后。** 二阶供应商收入可能滞后一到两个季度才体现。
5. **忽略供给反应。** 如果供给能快速增加，瓶颈不一定带来持久利润。
6. **只看赢家不看输家。** 一个主题可能同时创造受益链和受损链。
7. **跳过公司级研究。** 雷达候选不是买入结论，必须进入深研。
8. **把主题正确等同于股票便宜。** 行业传导正确，也可能因为估值太贵而不是好投资。
