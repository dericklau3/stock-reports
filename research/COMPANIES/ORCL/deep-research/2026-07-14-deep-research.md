# Oracle（ORCL）深度研究更新 — 2026-07-14

- **Research view**: 建设性但需观察；估值吸引力较 2026-06-27 提高
- **Confidence**: 中等
- **Time horizon**: 12–24 个月
- **Main thesis**: Oracle 正从高粘性数据库与企业软件现金牛，转型为“软件现金牛 + AI 云基础设施融资平台”。OCI 与 RPO 证明需求强劲，但资本开支、债务、租赁承诺、客户集中与普通股稀释决定最终每股回报。
- **Key risk**: Oracle 可能为少数大型 AI 客户提前锁定过多数据中心、电力、GPU、债务和长期租赁义务；即使收入增长兑现，若合同利润率或客户履约低于预期，每股自由现金流仍可能长期不佳。

> 本报告是研究支持，不是个性化投资建议。市场价格采用 2026-07-13 收盘价；财务数据以 Oracle FY2026 Form 10-K、Q4/FY2026 8-K 及其 Exhibit 99.1 为主。2026-06-27 之后截至本次核验，没有新的 Oracle 10-Q、10-K 或 8-K，因此本次是“价格、估值与风险事件更新”，不是新财报后的基本面重写。

## 1. Executive View

### 结论先行

**ORCL 在 131.54 美元已经进入“可接受的小仓分批研究区”，但还没有进入无条件的强安全边际区。**

与 2026-06-27 报告相比：

- 股价从 148.53 美元降至 131.54 美元，跌幅约 11.4%。
- 按 FY2027 non-GAAP EPS 指引 8.05 美元，forward non-GAAP P/E 从约 18.5 倍降至约 16.3 倍。
- 按当前股价和 2026-06-12 流通股数估算，市值约 3,789 亿美元。
- 沿用 6 月 26 日市场数据中的 EV 与市值差额桥接，当前粗略 EV 约 5,144 亿美元，对 FY2027 900 亿美元收入指引约 5.7 倍 EV/Sales。
- SEC submissions 显示 2026-06-26 之后没有新的重大申报，因此股价下跌主要反映市场继续重估融资、租赁、客户集中与资本回报风险，而不是新财报下调。

### 为什么更便宜，但仍不能只看 P/E

16.3 倍 FY2027 指引 EPS 对一家公司指引收入增长约 34% 看起来很便宜。但这个 EPS 口径有四个限制：

1. 是 non-GAAP 指标，SBC 是真实经济成本。
2. Oracle FY2026 自由现金流为负 236.86 亿美元。
3. FY2027 仍预计需要约 400 亿美元融资，包括 200 亿美元 ATM 普通股计划。
4. 2,600 亿美元额外数据中心租赁承诺尚未全部进表，会在未来多年形成固定成本。

因此，ORCL 当前真正的估值问题不是“16 倍 P/E 是否便宜”，而是：

> FY2027 的 8.05 美元 EPS 和 900 亿美元收入，能否在可接受的债务、租赁、稀释和长期 FCF 回报下实现？

### 核心支持因素

- FY2026 revenue 673.57 亿美元，同比增长 17%。
- FY2026 cloud revenue 339.89 亿美元，同比增长 39%。
- FY2026 OCI revenue 181.01 亿美元，同比增长 77%。
- Q4 OCI revenue 57.87 亿美元，同比增长 93%。
- RPO 达 6,380 亿美元，同比增长 363%。
- Oracle 数据库和软件支持有强转换成本，是扩张期的现金流底座。
- Multicloud Database 让 Oracle 可以在 Azure、Google Cloud 与 AWS 环境中继续变现数据库资产。
- 大型 AI 合同中，客户预付或自行提供硬件部分累计约 750 亿美元，可降低 Oracle 的直接资金负担。

### 最强反方理由

RPO 的绝对规模巨大，但 Oracle 没有充分披露客户集中度、各合同利润率、取消或延期条款及客户信用结构。2026 年 7 月出现的投资者诉讼进一步质疑 Oracle 是否充分披露 OpenAI 相关信用与融资风险；这些目前只是原告指控，不是法院认定，但说明市场的核心争议已从“有没有订单”转向“订单由谁承担信用和资本风险”。

## 2. Oracle 到底卖什么，怎么赚钱

### 初学者版

Oracle 帮企业把最关键的数据和业务流程稳定地运行起来。

| 业务 | 白话解释 | 谁付钱 | 经济特征 |
|---|---|---|---|
| 数据库与软件支持 | 存储和处理银行交易、企业订单、库存、财务及其他关键数据 | 银行、政府、电信、制造、零售及大型企业 | 高转换成本、高粘性、低增长现金牛 |
| 云应用 SaaS | Fusion、NetSuite、Oracle Health 等在线财务、人力、供应链和医疗软件 | 企业 CFO、人力部门、供应链部门、医院 | 经常性订阅、资本较轻、增速中等 |
| OCI 云基础设施 | 出租 GPU、服务器、存储、网络、电力和数据库云能力 | AI 公司、软件公司、大型企业和政府 | 高增长，但重资本、需提前建设产能 |
| 硬件与服务 | Exadata、服务器、实施、咨询和支持 | Oracle 大客户 | 完善解决方案，但不是主要估值引擎 |

### 今天的利润引擎

传统 Oracle Database、软件支持、Fusion/NetSuite 等业务依靠订阅、支持与许可收费。客户迁移关键数据库需要改代码、迁数据、重新审计、培训员工并承担停机风险，因此续费较稳定。

### 未来增长引擎

OCI 通过出租 AI 算力、网络、存储和数据库服务收费。Oracle 必须先承担服务器/GPU、数据中心、电力、网络、租赁和融资成本，再随客户使用逐步确认收入。

OCI 的经济公式可以简化为：

`客户云收入 − GPU/服务器折旧 − 数据中心租赁 − 电力 − 网络 − 运维 − 融资成本`

核心问题不是收入能否增长，而是成熟产能的税后资本回报率能否高于债务、租赁和股权成本。

### 关键术语

| 术语 | 含义 |
|---|---|
| OCI | Oracle Cloud Infrastructure，Oracle 的云服务器、GPU、存储、网络与数据库平台。 |
| RPO | 已签约但尚未确认的收入义务；它是收入线索，不等于现金和利润。 |
| Capex | 购买 GPU、服务器、网络设备及建设数据中心的资本支出。 |
| FCF | 自由现金流，本文按经营现金流减资本支出理解。 |
| Multicloud | 客户在 AWS、Azure 或 Google Cloud 中使用 Oracle Database 服务。 |
| Customer prepayment | 客户先付款，Oracle 后续交付服务。 |
| Customer-supplied GPU | 客户自行提供 GPU，Oracle 提供数据中心、电力、网络和运维。 |
| ATM issuance | 公司通过公开市场逐步增发普通股，降低债务压力但稀释现有股东。 |

## 3. 产品与分部分析

### FY2026 收入结构

| 产品类别 | FY2026 收入 | 同比 | 占总收入 |
|---|---:|---:|---:|
| Cloud | 339.89 亿美元 | +39% | 50.5% |
| Software | 245.41 亿美元 | -1% | 36.4% |
| Hardware | 30.84 亿美元 | +5% | 4.6% |
| Services | 57.43 亿美元 | +10% | 8.5% |
| Total | 673.57 亿美元 | +17% | 100% |

### OCI：增长最快，也是资本风险中心

FY2026 OCI revenue 为 181.01 亿美元，同比增长 77%；Q4 OCI revenue 为 57.87 亿美元，同比增长 93%。

优势：

- Oracle Database 存量客户提供天然迁云入口。
- 大型 GPU 集群与低延迟网络适合 AI 训练和推理。
- Multicloud 模式避免客户必须把全部工作负载迁到 OCI。
- 客户预付或自带 GPU 可减少 Oracle 的部分资本责任。

限制：

- AWS、Azure、GCP 的规模、生态与开发者基础更强。
- 大型 AI 客户合同集中，信用与议价风险高。
- 数据中心投产初期利用率低，折旧和租赁成本先发生。
- GPU 供给增加后，AI 算力可能从短缺转向价格竞争。

### Cloud Applications：稳定但不是爆发核心

FY2026 cloud applications revenue 为 158.88 亿美元，同比增长 11%。Fusion ERP/HCM/SCM、NetSuite 与 Oracle Health 带来经常性收入，但当前增速不足以独立支撑 AI 云估值。

### Software：护城河和现金牛

FY2026 software revenue 为 245.41 亿美元，同比下降 1%；其中 license 收入下降 9%，software support 增长 1%。

正面是支持收入仍稳定，说明存量数据库转换成本存在；负面是传统业务缺乏增长，新工作负载面临 PostgreSQL、Snowflake、Databricks、MongoDB 与云原生数据库竞争。

### SBC 与稀释

- FY2026 SBC 为 48.11 亿美元，占收入约 7.1%。
- FY2026 diluted weighted-average shares 为 29.14 亿股，同比增长约 2%。
- 2026-06-23 的 424B5 允许 Oracle 通过销售代理发行最多 200 亿美元普通股，并新增多家销售代理。
- FY2027 预计融资约 400 亿美元，包含 200 亿美元 ATM。
- Mandatory convertible preferred 最终也可能增加普通股股数。

结论：投资者应关注 fully diluted shares，而不是只看基本股数或 non-GAAP EPS。

## 5. 管理层与资本配置

Oracle 管理层在销售、数据库工程、成本控制和获取大型企业合同方面有长期执行记录。FY2026 OCI 与 RPO 增长说明其确实抓住了 AI 基础设施需求窗口。

但管理层当前承担的是 Oracle 历史级别的资本配置决策：

- 巨额 capex；
- 高债务；
- 长期租赁；
- 电力与采购承诺；
- ATM 普通股发行；
- 对少数大型 AI 客户的经济依赖。

后续可信度不能只看收入是否达标，还要看：

1. RPO 转收入速度。
2. 客户预付与自带 GPU 比例。
3. OCI 毛利率和成熟期资本回报。
4. FCF 收窄与转正时间表。
5. 债务、租赁承诺与 fully diluted shares。
6. 是否披露更充分的客户集中和信用风险。

## 6. 新增法律与披露风险

2026-07-02，Courthouse News 报道 Sterling Heights Police & Fire Retirement System 提起诉讼，指控 Oracle 在 2026 年 2 月约 250 亿美元债券发行文件中，没有充分披露与 OpenAI 相关的客户信用、收入目标和云计算付款能力风险。

必须严格区分：

- **已确认事实**：诉讼已被报道提出；Oracle 拒绝评论；原告寻求赔偿及撤销债券购买。
- **尚未确认的指控**：OpenAI 是否构成 RPO 的绝大部分、Oracle 是否掌握并遗漏了特定财务风险、披露是否违反证券法。
- **研究含义**：诉讼本身短期未必有重大财务影响，但强化了客户集中、信用风险与披露透明度问题，值得纳入风险折价。

截至本次核验，SEC submissions 在 2026-06-26 之后没有新的 Oracle 8-K、10-Q 或 10-K，因此不能把诉讼中的指控当作 Oracle 已承认或监管机构已认定的事实。

## 7. 竞争格局

| 领域 | Oracle 优势 | Oracle 弱点 |
|---|---|---|
| 企业数据库 | 存量客户、转换成本、Exadata、行业知识 | 开源与云原生数据库抢占新工作负载 |
| AI 云基础设施 | 大型 GPU 集群、低延迟网络、灵活客户硬件模式 | 规模与生态弱于 AWS/Azure/GCP |
| Multicloud | 可进入竞争对手云内变现 Oracle Database | 客户控制权可能继续留在合作云平台 |
| 企业应用 | Fusion、NetSuite、Oracle Health 与数据库协同 | SAP、Workday、Salesforce 等竞争强 |
| 医疗 | Cerner installed base 与临床数据 | 整合、客户体验和现代化仍需验证 |

Oracle 最强的护城河仍是数据库与企业关键工作负载，而不是 OCI 本身。OCI 的护城河必须用客户续约、价格纪律、利用率、利润率和资本回报来证明。

### 为什么使用情景 P/E

Oracle 仍有大量高毛利软件业务，P/E 有解释力；但 FCF 为负且资本结构变化快，所以必须让倍数反映资本强度和风险：

- Bear：收入或 RPO 转化不及预期，长期负 FCF和稀释使市场只给 12–14 倍 FY2027 non-GAAP EPS。
- Base：900 亿美元收入大致兑现，OCI 高增长但 FCF 尚未完全证明，市场给 18–22 倍。
- Bull：OCI 交付顺利、客户预付降低资本压力、FCF 路径清晰，市场给 25–30 倍。

### 熊 / 基准 / 牛情景

| 情景 | 假设 | 估值倍数 | 隐含价格 | 相对 131.54 美元 |
|---|---|---:|---:|---:|
| Bear | 指引下调或 RPO 延期；OCI margin 弱；FCF 长期为负；稀释扩大 | 12–14x × 8.05 | **97–113 美元** | **-26% 至 -14%** |
| Base | FY2027 收入接近 900 亿；OCI 高增长；融资可控但尚未证明 FCF 拐点 | 18–22x × 8.05 | **145–177 美元** | **+10% 至 +35%** |
| Bull | RPO 顺利转收入；客户预付/自带 GPU 降低现金投入；FY2028 后 FCF 明显改善 | 25–30x × 8.05 | **201–242 美元** | **+53% 至 +84%** |

这些是条件估值，不是确定目标价。最大局限是用 FY2027 non-GAAP EPS 作为基准，尚未完整反映未来折旧、租赁和股权稀释。因此实际仓位不能按 bull case 决定。

### 实用价格分区

| ORCL 价格 | 研究判断 |
|---|---|
| **≤110 美元** | 强安全边际候选区；接近 bear case，需要确认没有基本面断裂 |
| **110–135 美元** | 可接受观察/小仓分批区；赔率改善，但仍需持续验证 FCF 与客户风险 |
| **135–175 美元** | 基准公允价值区；需要 Q1/Q2 FY2027 达标来支撑 |
| **175–200 美元** | 偏贵区；需要更清晰的 OCI margin 与 FCF 拐点 |
| **>200 美元** | 依赖 bull case，不宜在证据不足时追价 |

**131.54 美元位于可接受区上沿。** 如果只看估值，已经可以研究性小仓分批；如果要求强安全边际，则仍应等待 110 美元附近，或等待下一季财报提供更强确认。

## 9. 催化剂与监控计划

近期催化剂：

- FY2027 Q1 revenue growth 是否达到 27%–29%。
- Q1 cloud revenue growth 是否达到 58%–64%。
- Q1 non-GAAP EPS 是否达到 1.72–1.76 美元。
- FY2027 900 亿美元 revenue 与 8.05 美元 EPS 指引是否维持。
- RPO 新增量、未来 12/36/60 个月收入确认节奏。
- 客户预付和 customer-supplied GPU 披露。
- ATM 发行进度、发行均价与 fully diluted shares。
- Capex、net cash outlay、债务、利息和 FCF。
- 新数据中心租赁承诺开始进表后的成本影响。
- Oracle Database@Azure/Google Cloud/AWS 使用量。
- OpenAI 相关诉讼进展及公司是否提供额外披露。

| 债务与利息成本 | 中高 | 高 | 高融资需求压低未来股东现金回报 |
| OCI 价格竞争 | 中 | 中高 | GPU 供给增加后，合同续约价格和利润率承压 |
| 传统软件衰退 | 中 | 中高 | 软件现金牛减弱会削弱基础设施扩张的下行保护 |
| 证券诉讼与披露风险 | 中 | 中 | 当前为未决指控，但可能推动更严格披露与法律成本 |
| 管理层执行复杂度 | 中 | 高 | 联席 CEO 和新 CFO 正执行公司历史最大资本计划 |

## 11. 四种投资风格压力测试

### 反偏见说明

- **信息丰富度**：A。
- **主要盲点**：大型 AI 合同的客户集中度、利润率、取消条款与资本责任没有充分披露。
- **聪明投资者最可能分歧之处**：有人把负 FCF 视为高回报建设期，有人把它视为 Oracle 永久转向低回报重资产模式。

### Buffett-style：Needs further observation

- **关键问题**：Oracle 是否仍是高资本回报的软件公司，还是已成为依赖融资的基础设施公司？
- **支持证据**：数据库转换成本强，软件支持稳定，FY2026 OCF 319.77 亿美元。
- **反对证据**：FCF -236.86 亿美元，额外租赁承诺 2,600 亿美元，FY2027 仍需融资。
- **行动含义**：价格更合理，但要看到 OCI 成熟资本回报和 FCF 改善后才适合提高确定性。
- **五年停市问题**：如果市场关闭五年，Oracle 的数据库现金牛能否覆盖 OCI 的全部融资和固定成本？

### Munger-style：Needs further observation

主要失败路径：

| 路径 | 机制 | 后果 |
|---|---|---|
| 高估 RPO 质量 | 客户集中、确认慢、利润率低 | 收入故事成立但股东回报不成立 |
| 固定成本锁死 | 长期租赁、电力与采购承诺生效 | 需求下降时利润与现金流反向放大 |
| OpenAI 信用风险 | 关键客户资金能力低于市场预期 | RPO 延期、重议或损失 |
| 稀释超预期 | ATM、可转优先股与 SBC | 企业增长无法转化为每股增长 |
| 软件现金牛变弱 | 开源和云原生数据库侵蚀 | 失去扩张期下行保护 |

- **最大心理陷阱**：把“AI 需求很大”误等同于“Oracle 的每一美元 capex 都有高回报”。
- **三年后复盘问题**：如果投资失败，今天最明显的红旗很可能是 2,600 亿美元租赁承诺与客户集中度没有被充分定价。

### Duan Yongping-style：Needs further observation

- **生意本质**：Oracle 帮企业把不能出错的数据和流程稳定运行，并为迁移困难持续收费。
- **用户价值**：数据库与企业应用价值已被几十年验证；AI 云需求真实，但经济性尚未完整验证。
- **产品优势**：Oracle Database 品牌和转换成本强，OCI 品牌与生态弱于三大云平台。
- **人和文化**：销售与工程执行强，但资本纪律正在接受历史级压力测试。
- **价格判断**：131.54 美元已进入可接受区，但“对的价格”仍取决于 OCI 是否创造而非消耗每股价值。
- **追问**：OCI 扩张是在长期创造客户价值，还是用资产负债表追逐短期算力短缺？

### Li Lu-style：Needs further observation

- **能力圈**：数据库、软件支持与企业应用可理解；大型 AI 合同经济性因披露不足而超出高置信能力圈。
- **长期趋势**：企业 AI、数据中心与数据智能化是长期趋势，Oracle 位于企业数据与算力交叉点。
- **下行保护**：传统软件现金牛和 RPO 提供保护，但债务、租赁、负 FCF 与客户集中削弱保护。
- **研究深度结论**：值得持续研究，可在明确仓位纪律下小仓验证，但不适合仅凭低倍数做高置信重仓。
- **追问**：二十年后，Oracle 会是企业 AI 数据基础设施的关键节点，还是 AI capex 高峰期的过度扩张者？

## 12. 与用户交易纪律的匹配

### 使命漂移

Oracle 从数据库、企业软件扩张到云基础设施和 AI，仍围绕“帮助企业管理和运算关键数据”的核心使命。OCI、AI Database 与 Multicloud Database 属于强化核心能力，不构成明显使命漂移。

但如果 Oracle 的资本越来越主要用于为少数模型公司提供通用 GPU 容量，而数据库和企业应用协同不能证明，则公司可能从“企业数据平台”偏移为“高杠杆算力出租商”。这是需要监控的潜在使命稀释，而非当前已经确认的使命漂移。

### 组合分类

ORCL 应归入：

> 企业软件 / 云基础设施 / AI 基础设施混合类别，中高 beta，且资产负债表风险高于传统软件龙头。

与 MSFT、AMZN、GOOGL、VRT、GEV、NVDA 及其他 AI 基础设施持仓存在一定相关性。按照用户纪律：

- 同类资产总权重不应超过 15%。
- 买入后现金 + 低 beta / 核心股票仍需至少 30%。

本次没有可验证的完整当前持仓，因此不能确认组合健康度或具体可买仓位。股票本身进入可接受区，不等于组合层面自动合规。

## 13. 最终判断

### 是否值得继续研究

**值得。** Oracle 是 AI 基础设施周期中最需要同时研究订单质量、资本结构和软件护城河的标的之一。

### 当前是否可买

**股票本身：可以进入小仓分批研究区，但不是高确定性重仓区。**

- 当前 131.54 美元：可接受区上沿。
- 110 美元以下：安全边际更强，但需确认不是基本面断裂。
- 145–177 美元：接近基准公允价值，需要财报兑现。
- 200 美元以上：明显依赖 bull case。

### 是否适合长期持有

传统数据库、软件支持和企业应用具备长期持有属性；OCI 的高资本扩张使整体公司的长期持有条件变为：必须持续验证 RPO 质量、客户信用、利用率、利润率、FCF 和每股稀释。

### 最大风险

不是 AI 需求消失，而是 Oracle 为少数客户承担过多长期资本责任，最终得到高速收入但低每股 FCF 回报。

## What would change my view

上调至 **Positive**：

- FY2027 Q1/Q2 收入和 cloud growth 达到指引上半区。
- RPO 按披露节奏转收入，客户预付比例提高。
- OCI margin 和数据中心利用率改善。
- FCF 明显收窄，融资和 ATM 稀释低于预期。
- 大客户信用与合同结构得到更透明披露。

下调至 **Neutral / Negative**：

- FY2027 900 亿美元收入或 8.05 美元 EPS 指引下调。
- RPO 延期、重议或客户信用风险显性化。
- 证券诉讼披露出此前未被市场理解的重大客户集中风险。
- 租赁、电力和采购承诺继续大幅增加。
- FCF 恶化、债务或股权融资显著超预期。
- 传统 software support 开始持续下降。
